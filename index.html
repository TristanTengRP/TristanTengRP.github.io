PRCE回溯

2020-04-29

之前针对/^xxx$/的正则匹配，介绍过可以使用%0a绕过；今天通过一道题目看了下P神的PREC回溯绕过正则匹配的方法，就记录学习一下；

回溯过程

首先了解一下回溯过程，当有一个正则为/<?.*[(`;?>]./。假设匹配的输入是<?php phpinfo();//aaaaa，实际匹配的流程是这样的：

(接用下P神的图)
当匹配到.\时，会最终匹配到字符串结尾。但因为对于正则表达式来说，应该还要匹配一个字符[(`;?>];所以开始回溯，从后往前，先吐出一个a，发现不匹配，继续吐a….直到吐出;后，匹配上了，于是继续下一个匹配，即.*，匹配到结尾。到这匹配结束；
我们可以发现，这里总共回溯了8次；
而PHP为了给PCRE设定了一个回溯次数上限；当超过回溯次数后，就会返回false。


于是我们就可以利用这一点，进行正则表达式绕过；

实例演示

FBCTF——RCEService
题目要求输入json格式，首先尝试{“cmd”:”ls”},回显结果index.php；但有很多常用命令被过滤；这时可以在我们的命令后加上1000000个a从而绕过正则匹配，达到命令执行的效果；

res = requests.post("http://af72594c-dbfc-4ef9-baa3-0738dbb5fdb9.node3.buuoj.cn/&quot;", data={"cmd":payload})

无列名注入——比较法盲注

在之前的一篇文章里，曾经介绍过使用union+select的方式进行无列名注入；今天再介绍另外一种无列名注入的方法——使用select +比较法进行盲注，从而实现无列名注入。
这种方法主要借助子查询并与查询结果进行按位比较；根据返回结果，从而盲注出数据内容；下图是我创建的一张表：

先简单介绍一下数据库是如何按位比较的：
先比第一位，如果相等则比第二位，以此类推；在某一位上，如果前者的ASCII大，不管总长度如何，ASCII大的则大，这个不难懂，和c语言的strcmp()函数原理一样，举几个例子：

glag > flag{bbbbb}
alag{zzzzzzzzzzz} < flag{bbbbb}
a < flag{bbbbb}
z > flag{bbbbb}

那么在保证第一列相等的前提下，即可使用按位比较，推测出第二列的内容：
当输入为’f’时，返回的是0；当输入为’g’的时候为1，可以知道内容首位为’f’；因此可以推测出剩余内容；

或许有人问如何保证第一列相等，方法是与上面一样的，依次猜测；
下面以GXYCTF中的一道题目来实践一下这种方法：

猜测注入类型

先测试一下，发现过滤了 union select, or 等等，报错注入也没用。输入 1&&1=1 和输入1&&1=2 时，发现回显不同，所以存在布尔盲注。

bypass information_schema

因为过滤了or,in所以不能使用information_schema和Innodb引擎查找；在之前那篇文章也介绍了几种bypass的方法，这里可以使用sys.x$schema_flattened_keys进行绕过

三月份参加校队考核的时候，遇到过无列名注入+bypass information_schema这个知识点，今天想做个记录（其实是填坑啦…）

知识点

Bypass information_schema

就Sql注入来说，我们对information_schema这个库自然非常熟悉，他是Mysql5.0以上版本自带的数据库，包含着Mysql中所有数据库的信息，包括（table_schema、table_name、column_name等）。可以形象的理解为“数据库字典”。我们经常使用这个数据库爆出库名、表名、字段名；正因如此，有些waf会过滤“or”，这样我们既使用不了order by，同时也查询不到Information__schema这个库，自然得不到相应的信息。今天就来总结一下，如何在查询不了information_schema的情况下，依然能从数据库中得到信息；

一、使用innodb引擎绕过
*注：这个方法并不适用所有情况，因为Mysql是默认关闭innodb引擎的；

既然无法查询information_schema库，就想办法能不能通过其他库进行相关信息查询；对于Mysql来说，可以使用innodb引擎实现。
在Mysql中，存储数据的默认引擎分为两类。一类是在5.5.x之前的MyISAM数据存储引擎，另一类是5.5.x版本后的innodb引擎。并且mysql开发团队在5.5.x版本后将innodb作为数据库的默认引擎。

而在mysql 5.6.x版本起，innodb增添了两个新表，一个是innodb_index_stats,另一个是innodb_table_stats。查阅官方文档，其对这两个新表的解释如下图：

从官方文档我们可以发现两个有用的信息：

1、从5.6.x版本开始，innodb_index_stats和innodb_table_stats数据表是自动设置的。
2、两个表都会存储数据库和对应的数据表。

先来了解一下这两个是什么东西：

schema_auto_increment_columns，该视图的作用简单来说就是用来对表自增ID的监控。

所以查询schema_auto_increment_columns只能查询含有自增字段的表名；

无列名注入、Join绕过逗号

虽然用以上方法，可以解决查询表的问题。但依然查询不到表的相关信息，这时就可以使用无列名注入；通过对要查询的列起别名，并查询这个别名将数据查出，达到未知列名，依旧查询到信息的效果；
可以使用

什么是无参RCE？

顾名思义，无参RCE就是通过传入没有参数的函数达到命令执行的效果；一般会出现这样的正则匹配时，考虑使用无参RCE

if(';' === preg_replace('/[^\W]+\((?R)?\)/', '', $_GET['code'])) {    
eval($_GET['code']);
}

面对这样的过滤，我们的函数是传递不了参数的，即只能使用这样的形式：

eval(a())    可以
eval(a('111'))     不可以

说白了，无参RCE有点类似套娃…前一个函数的结果作为下一个函数的参数；
既然我们不可以传递参数，那么只能使用PHP提供的函数作为我们的参数，达到执行效果；下面介绍几个PHP无参RCE经常使用的函数和方法：

get_defined_vars()

get_defined_vars()返回所有变量名及其对应的变量名；

操纵数组的函数

end() – 将内部指针指向数组中的最后一个元素，并输出
next() – 将内部指针指向数组中的下一个元素，并输出
prev() – 将内部指针指向数组中的上一个元素，并输出
reset() – 将内部指针指向数组中的第一个元素，并输出
each() – 返回当前元素的键名和键值，并将内部指针向前移动


array_rand(array_flip())

array_rand()随机返回数组中一个元素的Key值；
array_flip()将数组中的元素，键值位置进行对调；

比如我们想要读取数组中的flag.php；因为array_rand()只能返回数组元素中的key，所以可以使用array_flip()将键值位置对调，再结合array_rand()读取flag.php；因为array_rand()是随机返回，所以需要多试几次；

session_id(session_start())


session_id()可以获取PHPSESSID的值，而我们知道PHPSESSID允许字母和数字出现，那么我们就有了新的思路，即hex2bin，可以将我们要进行的操作经过hex()存放到PHPSESSID中。
使用session之前需要通过session_start()告诉PHP使用session，php默认是不主动使用session的。

getallheaders()获取所有http头部信息
所以可以通过添加http头部信息,达到RCE的效果

什么是序列化与反序列化？

首先，介绍一下什么是序列化与反序列化：php程序为了保存和转储对象，提供了序列化的方法，php序列化是为了在程序运行的过程中对对象进行转储而产生的。序列化可以将对象转换成字符串，但仅保留对象里的成员变量，不保留函数方法。反序列化则是将字符串还原为存储对象；
在PHP中，序列化函数为serialize,反序列化函数为unserialize；
举个例子：

<?php

class Test{

     public$a = 'ThisA';

     protected$b = 'ThisB';

     private$c = 'ThisC';

     publicfunction test1(){

              return'this is test1 ';

     }

}

$test = new Test();

var_dump(serialize($test));

?>

经过序列化之后的结果为：

string(84) "O:4:"Test":3:{s:1:"a";s:5:"ThisA";s:4:"*b";s:5:"ThisB";s:7:"Testc";s:5:"ThisC";}"

其中O代表的是对象（如果是A，则代表是数组），4表示该对象名称有4个字符；“Test”表示该对象的名称；:3表示该对象里有3个成员。接着是括号里面的。我们这个类的三个成员变量由于变量前的修饰不同，在序列化出来后显示的也不同。
因为第一个变量a是Public属性，所以序列化后为 s:1:”a”;s:5:”ThisA”; 从左往右看，第一个分号前表示对象属性为字符串a，属性长度为1；第二个分号前表示对象属性所对应的值为字符串“ThisA”，长度为5；
因为第二个变量b是protect属性，序列化后，对应属性变成*b,其实应该是%00*%00b
对于第三个变量c是private属性，序列化后，对应属性变成Testc，其实应该是%00类名%00c
从结果可以发现，序列化并不保存对象方法；因此反序列化还原为对象时，自然也不包括对象方法；

常见魔术方法

在了解完php序列化和反序列化的过程后，接下来介绍一下常见可能触发反序列化漏洞的魔术方法：

__wakeup() //使用unserialize时触发
__sleep() //使用serialize时触发
__destruct() //对象被销毁时触发
__call() //在对象上下文中调用不可访问的方法时触发
__callStatic() //在静态上下文中调用不可访问的方法时触发
__get() //用于从不可访问的属性读取数据
__set() //用于将数据写入不可访问的属性
__isset() //在不可访问的属性上调用isset()或empty()触发
__unset() //在不可访问的属性上使用unset()时触发
__toString() //把类当作字符串使用时触发
__invoke() //当脚本尝试将对象调用为函数时触发

总体来说，要先绕过 1 – 5 五个过滤，其中 5 最好绕，而且是跟其他四个条件没关系的，直接对变量名传递数组即可。之后看 4，打开我们传递的$file变量文件，且里面的内容要等于给定值；想到 data://；但是直接绕过的话有两个问题，第一 debu 作为关键字被过滤条件 1 拦截，但是无所谓，可以使用 URL 编码绕过；这是因为$_SERVER[‘QUERY_STRING’]在传递变量后，并没有进行URLDecode，而GET[ ]会进行urldecode，所以只要将我们的变量进行Urlencode传入，绕过对$_SERVER[query_string]的正则匹配；

先来讲解一下create_function()的基本原理：
create_function()主要用来创建匿名函数,有时候匿名函数可以发挥它的作用。

string create_function    ( string $args   , string $code   )

举个栗子：

create_function('$fname','echo "zhang".$fname')

上面这个匿名函数的作用等同于：

function aa($fname) 
{
echo "Zhang".$fname.;
}

真本该是一个利于开发者的操作，但如果对$arg和$code不进行过滤控制，则可能导致代码注入；
再举一个栗子：
如果$fname=;}phpinfo();//
那么上面的执行函数就会变成这样：

function aa($fname)
{
    echo "zhang";}phpinfo();//
}

格式调整一下：
function aa($fname)
{
    echo "zhang";}
    phpinfo();//}

成功使用}闭合函数aa，并使得我们构造的phpinfo()函数逃逸，成功执行，“//”主要是将原本函数aa的大括号注释，保证php语法正确；
因此，我们完全可以在这块，上传一句话木马拿shell；这就是Create_function的代码注入问题基本原理；
